## 콜 스택

- 함수 호출 시 시스템이 자동으로 사용하는 스택
- 재귀 DFS 같은 코드에서 자연스럽게 사용됨

## 명시적 스택

- 개발자가 자료구조로 직접 만드는 스택
- 반복문 DFS에서 자주 사용

## 차이

### 콜 스택
- 코드가 간결함
- 깊이가 깊으면 스택 오버플로 위험

### 명시적 스택
- 제어가 더 명확함
- 메모리 사용과 처리 순서를 직접 다루기 쉬움

## 면접 포인트

- 재귀 DFS를 반복 DFS로 바꾼다는 것은, 암묵적 콜 스택을 명시적 자료구조로 드러내는 것이라고 설명할 수 있습니다.
