## 파일 시스템이란?

파일 시스템은 디스크에 파일과 디렉터리를 어떻게 저장하고 관리할지 정한 구조입니다.

대표 예시:
- Windows: NTFS
- Linux: ext4, XFS
- 고급 기능 중심: ZFS

## NTFS와 ZFS

### NTFS
- Windows 기본 파일 시스템
- 권한, 저널링, 압축, 암호화 등을 지원

### ZFS
- 파일 시스템과 볼륨 관리 기능을 함께 제공
- 스냅샷, 체크섬, 복구, RAID 유사 기능 등 강력한 데이터 무결성 기능이 특징

## i-node란?

리눅스 계열 파일 시스템에서 파일의 메타데이터를 저장하는 구조입니다.
즉, "파일 이름"이 아니라 **파일 자체의 정보**를 담고 있습니다.

## i-node에 담기는 정보

- 파일 타입
- 권한(permission)
- 소유자 UID/GID
- 파일 크기
- 생성/수정/접근 시간
- 링크 수
- 데이터 블록 위치 포인터

## i-node에 없는 것

- 파일 이름

파일 이름은 디렉터리 엔트리에서 "이름 -> i-node 번호" 형태로 연결됩니다.

## 왜 중요한가?

- 하드 링크를 설명할 수 있음
- "삭제했는데 프로세스가 파일을 계속 잡고 있다" 같은 상황을 이해할 수 있음
- 디스크 사용량과 메타데이터 구조를 설명하기 쉬움

## 면접 포인트

- i-node는 파일 이름이 아니라 메타데이터와 데이터 위치를 가진다고 설명하면 핵심을 잘 짚은 것입니다.
- 하드 링크와 소프트 링크 차이까지 연결하면 더 좋습니다.
