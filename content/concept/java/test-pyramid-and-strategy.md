## 테스트 피라미드

아래층일수록 더 많이, 위층일수록 더 적게 두자는 전략입니다.

- 많은 단위 테스트
- 적당한 통합 테스트
- 소수의 E2E 테스트

## 왜 이런 구조인가?

- 단위 테스트는 빠르고 싸다
- E2E는 느리고 깨지기 쉽다

## 현실적인 전략

- 핵심 비즈니스 로직은 단위 테스트
- DB/외부 연동은 통합 테스트
- 핵심 사용자 시나리오는 E2E

## 면접 포인트

- 테스트 종류를 다 많이 쓰는 것이 아니라, 비용 대비 효과를 고려한 배치가 중요합니다.
