## IoC란?

객체 생성과 의존성 관리를 애플리케이션 코드가 직접 하지 않고, 컨테이너가 담당하는 개념입니다.

## DI란?

의존 객체를 외부에서 주입받는 방식입니다.
Spring은 생성자 주입, 필드 주입, 세터 주입 등을 지원합니다.

## 왜 필요한가?

- 객체 간 결합도를 낮춤
- 테스트가 쉬워짐
- 구현 교체가 쉬워짐

## Bean 생명주기

일반적으로:
1. Bean 생성
2. 의존성 주입
3. 초기화 콜백 수행
4. 사용
5. 컨테이너 종료 시 소멸 콜백 수행

## 면접 포인트

- IoC는 큰 개념, DI는 그 구현 방식 중 하나로 설명하면 좋습니다.
