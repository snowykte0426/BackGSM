## 프록시 객체가 왜 필요한가?

Hibernate는 지연 로딩을 위해 실제 엔티티 대신 프록시 객체를 먼저 반환할 수 있습니다.
즉, 당장 데이터가 꼭 필요하지 않다면 실제 조회를 늦추는 방식입니다.

## 어떤 문제가 생길 수 있나?

### 1. 준영속 상태에서 초기화 문제
- 영속성 컨텍스트가 닫힌 뒤 프록시 내부 데이터를 읽으려 하면 예외가 날 수 있습니다.

### 2. 타입 비교 문제
- 프록시는 실제 엔티티를 감싼 다른 런타임 객체일 수 있어
- 단순 클래스 비교나 캐스팅이 기대와 다르게 동작할 수 있습니다.

### 3. 의도하지 않은 추가 쿼리
- 프록시 접근 시점에 SQL이 나가므로 성능 문제가 숨겨질 수 있습니다.

## 실무에서 주의할 점

- 조회 시 fetch join, DTO projection 등을 적절히 사용
- 트랜잭션 경계 밖에서 엔티티 그래프를 마구 탐색하지 않기
- 프록시 초기화 시점과 쿼리 수를 항상 의식하기

## 면접 포인트

- 프록시는 편의 기능이지만, 지연 로딩과 성능 문제의 출발점이 되기도 한다는 점을 설명하면 좋습니다.
