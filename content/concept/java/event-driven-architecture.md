## EDA란?

이벤트를 중심으로 서비스나 컴포넌트가 느슨하게 연결되는 구조입니다.

예:
- 주문 생성 이벤트 발행
- 결제/알림/통계 서비스가 각각 구독

## 장점

- 느슨한 결합
- 확장성
- 비동기 처리에 적합

## 단점

- 흐름 추적이 어려울 수 있음
- 정합성 보장이 복잡해짐
- 장애 분석 난이도 상승

## 면접 포인트

- EDA는 확장성과 유연성을 주지만, 디버깅과 운영 복잡도가 커진다고 설명하면 좋습니다.
