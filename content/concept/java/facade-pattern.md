## 퍼사드 패턴이란?

복잡한 하위 시스템을 감싸고, 단순한 진입점을 제공하는 패턴입니다.

## 왜 쓰나?

- 호출 측 복잡도 감소
- 하위 시스템 변경 영향 축소

## 예시

- 여러 외부 API를 묶는 서비스
- 복잡한 내부 모듈을 단순 인터페이스로 제공

## 면접 포인트

- 퍼사드는 "단순한 진입점 제공"이 핵심입니다.
