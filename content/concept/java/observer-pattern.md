## 옵저버 패턴이란?

한 객체의 상태 변화가 있을 때, 이를 구독하는 여러 객체에게 알리는 패턴입니다.

## 예시

- 이벤트 리스너
- UI 상태 변경 알림
- 도메인 이벤트 발행

## 장점

- 느슨한 결합
- 확장성

## 단점

- 흐름 추적이 어려울 수 있음
- 옵저버가 많아지면 복잡도 증가

## 면접 포인트

- 이벤트 기반 설계와 연결해서 설명하면 좋습니다.
