## 모놀리식

하나의 애플리케이션으로 묶여 배포되는 구조입니다.

### 장점
- 단순함
- 개발/배포/디버깅이 비교적 쉬움
- 초기 속도가 빠름

### 단점
- 규모가 커지면 빌드/배포 부담 증가
- 특정 모듈만 독립 확장하기 어려움

## MSA

기능을 여러 독립 서비스로 나눠 운영하는 구조입니다.

### 장점
- 서비스별 독립 배포
- 기술 선택 유연성
- 팀 단위 분리 가능

### 단점
- 운영 복잡도 증가
- 분산 트랜잭션, 모니터링, 네트워크 장애 대응 필요

## 면접 포인트

- MSA가 무조건 더 좋은 것이 아니라, 조직 규모와 복잡성 감당 능력까지 같이 봐야 한다고 설명하면 좋습니다.
