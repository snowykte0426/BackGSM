## 테스트 더블이란?

실제 협력 객체 대신 테스트를 위해 사용하는 대체 객체입니다.

## 종류

### Dummy
- 자리만 채우는 객체

### Stub
- 미리 준비한 값을 반환

### Fake
- 단순하지만 동작하는 구현체
- 예: 인메모리 저장소

### Mock
- 호출 여부와 상호작용을 검증

### Spy
- 실제 객체를 감싸면서 일부 동작을 관찰

## 면접 포인트

- Mock과 Stub을 구분해서 설명할 수 있으면 좋습니다.
