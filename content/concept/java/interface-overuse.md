## 인터페이스가 항상 좋은가?

아닙니다.
인터페이스는 결합도를 낮추고 테스트를 쉽게 할 수 있지만,
의미 없는 추상화는 오히려 코드 이해 비용만 높입니다.

## 왜 문제인가?

- 구현체가 하나뿐인데 인터페이스부터 만드는 습관
- 파일 수 증가
- 코드 탐색 비용 증가
- 팀원이 "왜 추상화했는지" 이해하기 어려움

## 언제 필요한가?

- 구현체 교체 가능성이 큼
- 테스트 더블 주입이 중요함
- 외부 시스템 의존성 분리가 필요함

## 면접 포인트

- "무조건 인터페이스"보다, "변경 가능성과 경계 분리 필요가 있을 때 인터페이스"가 더 실무적인 답변입니다.
