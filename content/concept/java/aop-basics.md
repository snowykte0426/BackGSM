## AOP란?

AOP(Aspect Oriented Programming)는 공통 관심사를 핵심 비즈니스 로직과 분리해서 모듈화하는 방식입니다.

## 왜 필요한가?

로깅, 트랜잭션, 보안, 성능 측정 같은 기능은 여러 곳에 반복되기 쉽습니다.
이런 코드를 각 서비스마다 직접 넣으면 중복과 결합도가 커집니다.

## 핵심 아이디어

- 핵심 관심사: 실제 비즈니스 로직
- 공통 관심사: 여러 곳에 걸쳐 반복되는 로직

## 면접 포인트

- AOP는 "비즈니스 로직과 부가 로직을 분리하는 방법"이라고 설명하면 좋습니다.
