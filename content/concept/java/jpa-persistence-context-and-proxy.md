## 영속성 컨텍스트란?

엔티티를 관리하는 1차 캐시성 컨텍스트입니다.
같은 트랜잭션 안에서 같은 엔티티를 반복 조회하면 동일성 보장에 도움을 줍니다.

## 엔티티 생명주기

- 비영속(new)
- 영속(managed)
- 준영속(detached)
- 삭제(removed)

## 1차 캐시와 2차 캐시

### 1차 캐시
- 영속성 컨텍스트 단위
- 기본 제공
- 트랜잭션/세션 범위

### 2차 캐시
- 애플리케이션 범위에서 공유 가능
- 별도 설정 필요
- 캐시 일관성 전략이 중요

## 프록시 객체

지연 로딩을 위해 실제 엔티티 대신 프록시 객체를 반환할 수 있습니다.
필요한 시점에 실제 데이터를 로딩합니다.

## 왜 `instanceof`, 캐스팅 문제가 생길까?

- 프록시는 실제 엔티티를 상속하거나 감싼 별도 클래스일 수 있음
- 프록시 초기화 전/후 타입 비교가 기대와 다를 수 있음
- 준영속 상태에서 프록시를 건드리면 `LazyInitializationException`도 발생 가능

## OSIV의 문제

OSIV(Open Session In View)는 뷰 렌더링까지 영속성 컨텍스트를 열어 지연 로딩을 쉽게 해주지만,
트랜잭션 경계가 흐려지고 예기치 않은 쿼리가 뷰 계층에서 발생할 수 있습니다.

## 면접 포인트

- 영속성 컨텍스트는 단순 캐시가 아니라 엔티티 상태를 관리하는 공간입니다.
- 프록시, 지연 로딩, OSIV를 연결해서 설명할 수 있으면 좋습니다.
