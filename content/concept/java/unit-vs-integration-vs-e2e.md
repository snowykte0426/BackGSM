## 단위 테스트

가장 작은 단위의 로직을 독립적으로 검증합니다.

예:
- 서비스 메서드
- 도메인 객체
- 유틸 함수

특징:
- 빠름
- 외부 의존성 최소화

## 통합 테스트

여러 컴포넌트가 함께 동작하는지 확인합니다.

예:
- 서비스 + 리포지토리
- API + DB

## E2E 테스트

실제 사용자 흐름 전체를 검증합니다.

예:
- 로그인 -> 주문 -> 결제

## 면접 포인트

- 단위 테스트는 빠른 피드백
- 통합 테스트는 실제 연결 검증
- E2E는 사용자 시나리오 검증

이렇게 목적을 구분해 설명하면 좋습니다.
