## OCP란?

확장에는 열려 있고, 수정에는 닫혀 있어야 한다는 원칙입니다.

## 의미

기능이 추가될 때 기존 코드를 계속 뜯어고치는 대신,
새 구현을 추가하는 방식으로 확장하는 것이 이상적이라는 뜻입니다.

## 예시

결제 방식이 늘어날 때:

- `if-else`로 카드/계좌/포인트를 계속 추가하는 대신
- `PaymentStrategy` 인터페이스와 구현체를 늘리는 방식

## 면접 포인트

- OCP는 "기존 코드 수정이 0이어야 한다"는 극단적 뜻이 아니라,
변경 영향을 줄이는 설계 방향으로 이해하는 것이 좋습니다.
