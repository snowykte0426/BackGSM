## CQRS란?

Command와 Query의 모델을 분리하는 아키텍처 패턴입니다.

## 왜 쓰나?

- 읽기와 쓰기의 요구사항이 크게 다를 때
- 조회 최적화가 매우 중요할 때
- 복잡한 도메인과 읽기 모델을 분리하고 싶을 때

## 장점

- 읽기 모델 최적화 가능
- 책임 분리 명확

## 단점

- 구조 복잡도 증가
- 데이터 동기화 문제 고려 필요

## 면접 포인트

- CQRS는 무조건 쓰는 패턴이 아니라, 읽기/쓰기 요구 차이가 클 때 선택하는 전략이라고 설명하면 좋습니다.
