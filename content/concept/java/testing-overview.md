## 테스트의 목적

테스트는 단순히 버그를 찾는 것만이 아니라,

- 변경 시 회귀 방지
- 코드 설계 검증
- 문서 역할
- 리팩터링 안정성 확보

를 위한 안전장치입니다.

## 테스트를 나누는 대표 기준

- 단위 테스트
- 통합 테스트
- 엔드투엔드 테스트

## 면접 포인트

- 테스트는 품질 활동이면서 동시에 설계 피드백 도구라는 점을 설명하면 좋습니다.
