## 핵심 아이디어

비즈니스 규칙이 프레임워크, DB, UI 같은 외부 요소에 의존하지 않도록 하는 구조입니다.

## 중심 원칙

- 의존성 방향은 항상 안쪽으로
- 바깥은 안쪽에 의존
- 안쪽은 바깥을 몰라야 함

## 장점

- 비즈니스 로직 보호
- 테스트 용이성
- 기술 교체 유연성

## 단점

- 초반 구조 설계 비용
- 추상화 과다 위험

## 면접 포인트

- 클린 아키텍처는 "코어 보호" 관점에서 설명하면 좋습니다.
