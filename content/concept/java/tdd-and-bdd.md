## TDD

Test Driven Development.

흐름:
1. 실패하는 테스트 작성
2. 테스트를 통과하는 최소 코드 작성
3. 리팩터링

핵심은 테스트를 나중에 붙이는 것이 아니라,
테스트를 통해 설계를 끌고 가는 것입니다.

## BDD

Behavior Driven Development.
행동과 시나리오 중심으로 시스템을 설명하고 검증하는 접근입니다.

예:
- Given
- When
- Then

## 차이

- TDD는 개발 리듬과 설계 피드백에 더 초점
- BDD는 사용자 행동과 비즈니스 시나리오 표현에 더 초점

## 면접 포인트

- 둘을 대립 개념으로 말하기보다, 초점이 다른 테스트 접근이라고 설명하는 것이 좋습니다.
