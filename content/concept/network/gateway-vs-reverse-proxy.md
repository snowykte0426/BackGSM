## Reverse Proxy

클라이언트 앞단에서 요청을 받아 내부 서버로 전달하는 프록시입니다.

## API Gateway

Reverse Proxy 역할을 포함할 수 있지만,
인증, 라우팅, rate limiting, API 정책 같은 API 관리 기능에 더 초점이 있습니다.

## 면접 포인트

- API Gateway는 더 "API 제품/정책" 중심, Reverse Proxy는 더 "트래픽 중개" 중심이라고 설명하면 좋습니다.
