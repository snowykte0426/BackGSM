## 쿠키

브라우저에 저장되는 작은 데이터입니다.
인증 자체가 아니라, 세션 ID나 설정값 같은 정보를 담는 저장 수단입니다.

## 세션

사용자 상태를 서버에 저장하고, 클라이언트는 보통 세션 ID만 쿠키로 전달합니다.

장점:
- 서버가 상태를 직접 통제 가능

단점:
- 서버 저장소 관리가 필요
- scale out 환경에선 세션 공유 전략이 필요

## JWT

서버가 별도 세션 저장 없이 토큰 자체에 정보를 담아 인증에 사용하는 방식입니다.

장점:
- 서버 상태 저장 부담 감소
- 분산 환경에 유리

단점:
- 토큰 탈취 시 위험
- 즉시 강제 로그아웃, 세밀한 폐기가 까다로울 수 있음

## 면접 포인트

- 쿠키는 저장 위치/전달 수단
- 세션과 JWT는 인증 상태 관리 방식이라는 점을 구분해야 합니다.
