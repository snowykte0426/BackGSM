## 401 Unauthorized

인증이 없거나, 인증이 유효하지 않은 경우에 사용합니다.

예:
- 액세스 토큰이 없음
- 토큰이 만료됨
- 세션이 유효하지 않음

즉, "너는 아직 인증되지 않았다"에 가깝습니다.

## 403 Forbidden

인증은 되었지만, 해당 자원에 접근할 권한이 없는 경우에 사용합니다.

예:
- 일반 사용자가 관리자 전용 API 호출
- 특정 리소스 접근 권한 부족

즉, "누군지는 알겠는데, 이건 못 한다"에 가깝습니다.

## 핵심 차이

- 401: 인증 문제
- 403: 인가 문제

## 면접 포인트

- 401과 403을 뒤집어 말하지 않는 것이 중요합니다.
- 인증(Authentication)과 인가(Authorization)를 함께 연결해 설명하면 좋습니다.
