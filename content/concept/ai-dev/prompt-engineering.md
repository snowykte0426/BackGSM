## 프롬프트 엔지니어링이란?

LLM의 출력 품질을 높이기 위해 **입력(프롬프트)을 설계·최적화**하는 기술입니다.

## 핵심 기법

### 1. Zero-shot Prompting

예시 없이 바로 지시합니다.

```
"다음 리뷰가 긍정인지 부정인지 분류하세요: '배송이 너무 느렸어요'"
```

### 2. Few-shot Prompting

예시를 몇 개 제공해 패턴을 학습시킵니다.

```
"긍정: 배송이 빠르고 품질이 좋아요."
"부정: 포장이 엉망이었어요."
"??: 생각보다 별로였어요."
```

### 3. Chain-of-Thought (CoT)

**"단계적으로 생각하세요"** 지시로 추론 과정을 유도합니다.

```
일반: "12 × 7 + 15는?"
CoT: "단계별로 계산하세요. 12 × 7 = 84, 84 + 15 = 99"
```

복잡한 수학·논리 문제에서 정확도가 크게 향상됩니다.

### 4. Self-Consistency

같은 문제를 여러 번 풀어 다수결로 최종 답을 결정합니다. CoT와 함께 사용 시 효과적입니다.

### 5. Role Prompting

페르소나를 부여해 특정 관점의 응답을 유도합니다.

```
"당신은 10년 경력의 시니어 백엔드 개발자입니다. 다음 코드를 리뷰하세요."
```

### 6. Structured Output

출력 형식을 명시합니다.

```
"결과를 JSON으로 반환하세요: { name, score, reason }"
```

## System Prompt vs User Prompt

| 구분 | 역할 | 예시 |
|------|------|------|
| **System Prompt** | AI 역할·규칙 정의 (영속적) | "당신은 친절한 코딩 튜터입니다" |
| **User Prompt** | 실제 요청 | "버블 정렬을 설명해줘" |

## 프롬프트 작성 원칙

1. **명확한 지시**: 모호한 표현 대신 구체적 목표 기술
2. **컨텍스트 제공**: 배경 정보, 대상 독자, 출력 형식 명시
3. **예시 포함**: Few-shot 예시로 패턴 제시
4. **제약 조건**: 길이 제한, 금지 사항, 언어 명시
5. **역할 부여**: 전문가 페르소나로 품질 향상

## 할루시네이션(Hallucination) 방지

- **"모르면 모른다고 하세요"** 명시
- RAG로 외부 사실 주입
- 검증 가능한 출처 요청
- 낮은 Temperature 설정 (결정론적 출력)
