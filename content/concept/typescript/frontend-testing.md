## 프론트엔드에서 무엇을 테스트하나?

- 유틸 함수
- 상태 변화
- 컴포넌트 렌더링
- 사용자 상호작용
- API 연동 흐름

## 좋은 테스트 방향

- 구현보다 사용자 관점 동작 검증
- 내부 state보단 화면 결과와 이벤트 결과 검증

## 면접 포인트

- 프론트엔드 테스트는 "DOM이 어떻게 생겼는지"보다 "사용자가 무엇을 할 수 있는지"에 더 초점을 두는 것이 좋습니다.
