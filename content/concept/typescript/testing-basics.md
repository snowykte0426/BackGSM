## TypeScript에서도 테스트가 중요한 이유

타입이 있다고 해서 로직 버그가 사라지는 것은 아닙니다.
TypeScript는 타입 안정성을 주지만, 실제 동작 검증은 테스트가 맡아야 합니다.

## 주로 무엇을 테스트하나?

- 유틸 함수
- 비즈니스 로직
- API 호출 래퍼
- React/Vue 컴포넌트
- 서버 핸들러

## 면접 포인트

- 타입 시스템과 테스트는 서로 대체 관계가 아니라 보완 관계라고 설명하면 좋습니다.
