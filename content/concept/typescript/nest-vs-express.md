## Express

- 매우 가볍고 자유도가 높은 Node.js 웹 프레임워크
- 구조를 개발자가 직접 정해야 함

## NestJS

- TypeScript 기반 구조화 프레임워크
- DI, 모듈, 데코레이터, 테스트 친화 구조를 제공

## 선택 관점

### Express
- 단순 서비스, 빠른 시작
- 자유도가 높음

### NestJS
- 대규모 서비스
- 팀 단위 구조화와 일관성이 중요할 때

## 면접 포인트

- 자유도와 구조화의 트레이드오프로 비교하면 좋습니다.
