## 배열(Array)

- 메모리에 연속적으로 저장
- 인덱스로 즉시 접근 가능
- 조회는 빠르지만 중간 삽입/삭제는 비쌀 수 있음

## 연결 리스트(Linked List)

- 각 노드가 다음 노드를 가리킴
- 메모리가 연속적일 필요 없음
- 중간 삽입/삭제는 포인터 조정으로 유리할 수 있음
- 임의 접근은 느림

## 선택 기준

### 배열이 적합한 경우
- 인덱스 접근이 많음
- 캐시 친화성이 중요함
- 원소 수 변화가 크지 않음

### 연결 리스트가 적합한 경우
- 특정 위치의 삽입/삭제가 매우 잦음
- 순차 접근 중심

## 면접 포인트

- 실제 프로덕션에서는 CPU 캐시 효율 때문에 배열 기반 구조가 더 자주 선택되는 경우가 많습니다.
- 단순 이론뿐 아니라 현실적인 성능 차이까지 언급하면 좋습니다.
