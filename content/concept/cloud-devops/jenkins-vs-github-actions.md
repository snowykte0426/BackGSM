## Jenkins

- 자유도와 확장성이 높음
- 자체 운영 부담이 큼

## GitHub Actions

- GitHub와 통합이 뛰어남
- 상대적으로 빠르게 시작 가능

## 선택 기준

- 복잡한 커스텀 파이프라인과 사내 인프라 통합: Jenkins
- GitHub 중심 워크플로와 빠른 도입: GitHub Actions

## 면접 포인트

- 둘 중 뭐가 더 좋다기보다, 운영 통제력과 편의성의 트레이드오프라고 설명하면 좋습니다.
